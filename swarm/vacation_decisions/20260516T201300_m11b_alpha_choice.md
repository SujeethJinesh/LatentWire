# Vacation Decision: M11b Alpha Choice

## Situation

The information-value queue restructuring specifies M11b as EMA-smoothed
protection with `alpha=0.3` and describes it as "best from M11." The completed
M11 checker reported `m11_alpha_0_5` as the highest median-recovery arm on the
Granite-Small 12-trace packet, though no alpha passed.

## Options Considered

1. Use `alpha=0.5` because it was the highest observed M11 arm.
2. Use `alpha=0.3` because the human explicitly specified it for M11b.
3. Sweep alpha again inside M11b.

## Decision

I chose option 2: freeze M11b at `alpha=0.3`.

## Why This Preserves the Scientific Question

M11b tests budget insufficiency, not alpha selection. Using the explicitly
authorized alpha avoids post-hoc tuning after observing M11 data. Sweeping
alpha again would conflate budget sensitivity with smoothing-rate selection.

## What Would Invalidate This Decision

If the human intended M11b to use the empirically best M11 arm rather than the
explicitly written `alpha=0.3`, the M11b preregistration should be reviewed
before running. No M11b runner has been invoked at the time this note is
authored.
