## Interpretation

This falsifies the naive LLM-produced cryptographic syndrome path on the
cheapest exact-ID smoke. The deterministic candidate-syndrome gate remains a
valid protocol/headroom result, but prompting a source LLM to compute a compact
cryptographic digest is not a credible one-month positive method.

The source-final-only row is deliberately high: if the source is allowed to
communicate the private record name/final answer, the task is solved. The
matched model-packet row staying flat at target-only shows that the source model
did not transform private evidence into the required rate-capped packet.

## Decision

Prune: "LLM computes cryptographic source syndrome from text prompt" as a live
method branch.

Keep: the source-private side-information framing and candidate-syndrome
decoder as a protocol lower bound and benchmark contract.

Next: pivot to a source-private tool/test-log handoff where the source packet
is naturally emitted from private execution evidence, then test the same
control suite.
