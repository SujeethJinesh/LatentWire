## Paper Contributions Being Kept

1. Fixed-byte source-private packet protocol with destructive controls.
2. Public-basis benchmark gates plus negative/falsification ladder.
3. Systems byte/exposure accounting against KV/cache transfer baselines.

The candidate fourth contribution is a positive learned connector. It is not
yet evidence-backed and should not be claimed until the next gate passes.

## Decisions

- Promote the packet protocol, benchmark gates, and byte/exposure accounting
  for COLM.
- Cut HellaSwag receiver-improvement claims and TinyLlama mean hidden/query
  connector claims.
- Treat the Llama/Phi source-choice senders as cut unless revived through a
  separately selected prompt/scoring/calibration branch.
- Make the next live method branch a target-loss tokenwise soft-prefix/query
  connector, not another per-choice mean-cache map.
- Block native speed/HBM/goodput claims until matched NVIDIA measurements are
  ingested.

## Next Exact Gate

Mac-local first step: implement an isolated ARC n32 target-loss soft-prefix
preflight using Qwen source features and a frozen target. The run must emit
matched, target-only, packet-only, source-free, zero-source, row-shuffled,
same-byte visible text, Qwen-substituted, and label-shuffled controls.

If the ARC preflight clears controls, widen to the paper-facing OpenBookQA 3B
train-only receiver gate. Pass condition: packet-plus-receiver beats
packet-only by at least `+0.005` with positive paired CI95 lower bound and
source-destroying controls.

## Lay Explanation

This turn did not try to make the model smarter. It organized the evidence so
reviewers can see which ideas are still alive and which ones failed. The next
experiment will teach a tiny translator to turn one model's internal clues into
soft hints the other model can directly use, while checking that the hints
really came from the source model and are not just a hidden answer copy.

## Reviewer Boundary

The connector novelty is only defensible if the transmitted object is
per-example, source-conditioned, rate-limited, and tested against destructive
controls. Static prefix tuning, query bottlenecks, C2C/KV cache sharing, QJL,
TurboQuant, and SAE/common-feature-space work are all mandatory boundaries and
baselines, not optional related work.
