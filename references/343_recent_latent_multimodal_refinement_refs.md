# Recent Latent / Multimodal Refinement References for LatentWire

Web check: 2026-04-21. This memo prioritizes primary sources from 2024-2026 that are directly useful for LatentWire ablations around latent reasoning, soft-token transport, iterative refinement, multimodal connectors, recurrent depth, and cross-model communication.

## Latent reasoning and soft-token transport

- **[Coconut: Training Large Language Models to Reason in a Continuous Latent Space](https://arxiv.org/abs/2412.06769)**. Core idea: feed the last hidden state back as a continuous thought instead of committing to a text token. `Test:` replace LatentWire’s one-shot bridge with a recurrent latent-thought loop and sweep recurrence depth. `Telemetry:` latent norm, stepwise cosine drift, answer entropy, tokens-to-answer, exact-match / pass@1. `Risk:` gains can come from extra compute, not from the latent representation itself.

- **[SoftCoT: Soft Chain-of-Thought for Efficient Reasoning with LLMs](https://arxiv.org/abs/2502.12134)**. Core idea: generate soft thought tokens with a lightweight assistant and project them into the backbone’s representation space. `Test:` compare hard textual bridge tokens vs soft thought vectors from the same teacher prompts under matched budget. `Telemetry:` projection cosine, soft-token entropy, teacher-student agreement, budget-normalized accuracy. `Risk:` any win may depend on the assistant model rather than the bridge interface.

- **[Soft Thinking: Unlocking the Reasoning Potential of LLMs in Continuous Concept Space](https://arxiv.org/abs/2505.15778)**. Core idea: replace discrete reasoning steps with probability-weighted mixtures of embeddings in a continuous concept space. `Test:` convert LatentWire’s bridge payload into soft mixtures and compare against nearest-neighbor tokenization and plain hidden-state passing. `Telemetry:` mixture entropy, reconstruction loss, calibration, reasoning-step length, robustness by prompt family. `Risk:` soft mixtures can look better only because they smooth uncertainty, not because they preserve usable semantics.

- **[Token Assorted: Mixing Latent and Text Tokens for Improved Language Model Reasoning](https://arxiv.org/abs/2502.03275)**. Core idea: split reasoning traces into latent tokens plus text tokens so only the semantically necessary part stays discrete. `Test:` sweep the latent/text ratio in LatentWire and hold total bytes fixed. `Telemetry:` latent-token rate, text-token rate, compression ratio, selected-step accuracy, failure mode by trace type. `Risk:` hybrid traces can improve apparent efficiency while silently increasing supervision complexity.

- **[Compress to Think, Decompress to Speak: Dual-Mode Reasoning in Transformers](https://openreview.net/forum?id=c9FF7JR8BM)**. Core idea: separate a compression phase from a decoding phase so the model can reason in a compact latent buffer and only verbalize at the end. `Test:` add an explicit compress-then-decompress schedule and compare it to always-on bridge injection. `Telemetry:` pre/post compression loss, latent buffer size, decode sharpness, final-answer consistency, latency. `Risk:` a compress/decompress split may simply be another way of spending more test-time compute.

- **[Teaching Pretrained Language Models to Think Deeper with Retrofitted Recurrence](https://openreview.net/forum?id=Oq3Xblt0x1)**. Core idea: retrofit pretrained non-recurrent models into depth-recurrent ones so they can spend more compute in latent space. `Test:` run the bridge for multiple internal recurrence steps before any token emission and compare with a non-recurrent control at the same wall-clock budget. `Telemetry:` recurrence count, gradient stability, hidden-state similarity across loops, accuracy vs FLOPs, TTFT. `Risk:` recurrent depth may help only on tasks with obvious extra search budget.

## Diffusion-style refinement and iterative belief updates

- **[Continuous Diffusion Model for Language Modeling](https://arxiv.org/abs/2502.11564)**. Core idea: model language in a continuous diffusion geometry so iterative refinement is not forced through discrete-state bottlenecks. `Test:` replace a single bridge repair pass with continuous denoising steps over the latent transport state. `Telemetry:` denoising step count, KL to final belief, convergence rate, token recovery rate, quality vs steps. `Risk:` continuous diffusion wins can vanish once you normalize for additional refinement passes.

- **[LaDiR: Latent Diffusion Enhances LLMs for Text Reasoning](https://openreview.net/forum?id=z5cPEZ4n6i)**. Core idea: encode reasoning into a structured latent space and refine it with blockwise latent diffusion. `Test:` group transported reasoning into blocks and compare blockwise denoising against tokenwise repair in LatentWire. `Telemetry:` block-level edit distance, refinement stability, per-block uncertainty, answer consistency, compute per pass. `Risk:` blockwise refinement may work only because it reduces exposure to token-level noise.

- **[Latent Refinement Decoding: Enhancing Diffusion-Based Language Models by Refining Belief States](https://arxiv.org/abs/2510.11052)**. Core idea: keep uncertain positions as belief states and finalize them only after feedback has sharpened the posterior. `Test:` preserve low-confidence bridge outputs as distributions over candidates, then finalize after one or two feedback rounds. `Telemetry:` belief entropy, finalization rate, rollback count, calibration error, latency/accuracy tradeoff. `Risk:` better calibration is not the same as better task performance.

- **[Multimodal Chain of Continuous Thought for Latent-Space Reasoning in Vision-Language Models](https://arxiv.org/abs/2508.12587)**. Core idea: iteratively refine a continuous thought vector in a joint visual-text latent space, optionally with multimodal latent attention. `Test:` add a multimodal latent-thought loop to LatentWire and compare base vs multimodal-attended variants on image-grounded tasks. `Telemetry:` cross-modal attention mass, visual-token retention, latent trajectory stability, MMMU/ScienceQA accuracy. `Risk:` multimodal gains may come from better vision alignment rather than from latent refinement itself.

## Multimodal projectors, adapters, and connectors

- **[PaLM2-VAdapter: Progressively Aligned Language Model Makes a Strong Vision-language Adapter](https://arxiv.org/abs/2402.10896)**. Core idea: staged alignment with a progressively adapted language-model bridge can beat a simple perceiver-resampler projector. `Test:` compare LatentWire’s direct projector against a staged adapter trained in phases with the same target-side budget. `Telemetry:` convergence speed, frozen-encoder drift, adapter rank utilization, VQA / captioning accuracy, parameter count. `Risk:` staged training may look stronger simply because it gives the adapter more optimization opportunities.

- **[DeCo: Decoupling Token Compression from Semantic Abstraction in Multimodal Large Language Models](https://arxiv.org/abs/2405.20985)**. Core idea: compress visual tokens at the patch level and let the LLM do semantic abstraction, avoiding a double-abstraction bottleneck. `Test:` compare a simple patch compressor against a query-token projector inside LatentWire. `Telemetry:` visual-token count, semantic-flow concentration, grounding accuracy, trainable parameters, convergence speed. `Risk:` simpler compression can help only if the downstream model already has enough abstraction capacity.

- **[MoDA: Modulation Adapter for Fine-Grained Visual Grounding in Instructional MLLMs](https://arxiv.org/abs/2506.01850)**. Core idea: use instruction-guided modulation masks to refine already aligned visual features before generation. `Test:` insert a language-conditioned modulation stage on top of LatentWire’s visual bridge and evaluate grounding-heavy prompts. `Telemetry:` modulation sparsity, instruction-token attention, localization accuracy, hallucination rate, answer faithfulness. `Risk:` gains on grounding tasks may not transfer to open-ended reasoning tasks.

- **[Spatial-Aware Efficient Projector for MLLMs via Multi-Layer Feature Aggregation](https://arxiv.org/abs/2410.10319)**. Core idea: preserve spatial structure while reducing visual tokens through multi-layer feature aggregation. `Test:` swap LatentWire’s flat pooling for a spatial-aware projector and compare equal-token budgets. `Telemetry:` token count, spatial overlap score, object-localization accuracy, downstream VQA accuracy. `Risk:` spatial preservation may matter only for tasks with explicit localization cues.

## Cross-model communication and cache sharing

- **[Communicating Activations Between Language Model Agents](https://arxiv.org/abs/2501.14082)**. Core idea: agents can exchange intermediate activations directly instead of serializing to text. `Test:` replace text handoff between source and target models with activation handoff at one or more layers. `Telemetry:` activation cosine, inter-agent agreement, compute per exchange, accuracy vs communication cost. `Risk:` activation communication may be brittle outside the model pairs and tasks used in the paper.

- **[Cache-to-Cache: Direct Semantic Communication Between Large Language Models](https://arxiv.org/abs/2510.03215)**. Core idea: project and fuse source KV caches into the target model so models communicate semantically without intermediate text. `Test:` compare text handoff, hidden-state handoff, and KV-cache handoff under matched bytes. `Telemetry:` transferred KV bytes, layer-gating rate, reuse fraction, latency, accuracy by task family. `Risk:` cache transfer can win on latency while still being highly model-pair-specific.

- **[KVComm: Enabling Efficient LLM Communication through Selective KV Sharing](https://arxiv.org/abs/2510.03346)**. Core idea: share only the most informative KV pairs across models using layer-wise importance selection. `Test:` add selective KV sharing to LatentWire and sweep the shared-layer budget. `Telemetry:` shared-layer fraction, attention-importance scores, offset error, reuse rate, TTFT, downstream accuracy. `Risk:` selective reuse can understate its cost if alignment or selection logic is not counted explicitly.

## Highest-priority LatentWire ablations

1. **Latent recurrence vs one-shot bridge.** Compare a recurrent continuous-thought bridge to the current single-pass transport at matched wall-clock budget.
2. **Soft-token transport vs hard tokenization.** Replace nearest-neighbor reconstruction with soft mixtures of embeddings and measure whether uncertainty smoothing helps or hurts.
3. **Blockwise refinement vs tokenwise repair.** Test diffusion-style block denoising on transported latents against flat repair passes.
4. **Spatial-aware projector vs flat projector.** On multimodal inputs, replace flat pooling with a spatial-aware connector and track grounding/hallucination effects.
5. **Activation/KV communication vs text handoff.** For multi-model settings, compare latent communication primitives against ordinary textual exchange under equal byte budgets.
