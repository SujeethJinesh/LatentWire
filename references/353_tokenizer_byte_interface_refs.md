# Tokenizer / Byte Interface References for LatentWire

Web check: 2026-04-21. This memo prioritizes primary sources on tokenizer/vocabulary interfaces, byte-level models, retokenization, vocabulary adaptation, token alignment, and cross-tokenizer distillation. The recurring LatentWire question is whether the bridge should operate on raw hidden states, token IDs, aligned spans, or a byte-level common ground.

## Primary sources

- **[Byte Latent Transformer: Patches Scale Better Than Tokens](https://arxiv.org/abs/2412.09871)**. `Core idea:` replace a fixed subword vocabulary with entropy-driven byte patches so the model can scale on raw bytes without tokenizer brittleness. `LatentWire ablation:` compare the current bridge on standard tokenized inputs vs a byte/patch bridge with the same downstream head and matched byte budget. `Telemetry:` bytes per example, patch-length distribution, entropy threshold, latency, OOV robustness, answer accuracy. `Claim risks:` byte-level gains can come from changed compute allocation or longer effective context, not from a better communication interface.

- **[Cross-Tokenizer LLM Distillation through a Byte-Level Interface](https://arxiv.org/abs/2604.07466)**. `Core idea:` convert teacher outputs to byte-level probabilities and distill through a shared byte decoder head when teacher and student tokenizers differ. `LatentWire ablation:` add a byte-level auxiliary head to the bridge and compare it with direct token overlap, span alignment, and no shared interface. `Telemetry:` byte-level KL, decoder calibration, span coverage, alignment failure rate, downstream accuracy. `Claim risks:` byte-level targets reduce mismatch but can still be sensitive to teacher decoding quirks and byte-span segmentation quality.

- **[DWA-KD: Dual-Space Weighting and Time-Warped Alignment for Cross-Tokenizer Knowledge Distillation](https://arxiv.org/abs/2602.21669)**. `Core idea:` use dual-space weighting plus Soft-DTW-style alignment to handle both vocabulary mismatch and length mismatch in cross-tokenizer KD. `LatentWire ablation:` compare fixed-position supervision with time-warped alignment between source and target traces, then test whether uncertainty weighting improves bridge fitting. `Telemetry:` alignment cost, warped-path length, token entropy, teacher-student disagreement, sequence-level accuracy. `Claim risks:` alignment machinery can hide true failures by making mismatched sequences look cleaner than they are.

- **[CTPD: Cross Tokenizer Preference Distillation](https://arxiv.org/abs/2601.11865)**. `Core idea:` transfer preference supervision across heterogeneous tokenizers using aligned character spans and a teacher-anchored reference objective. `LatentWire ablation:` if the bridge needs a preference objective, compare raw reward/logit supervision with span-aligned preference distillation and teacher-anchored reference loss. `Telemetry:` preference win rate, span alignment accuracy, KL to teacher policy, calibration, task success rate. `Claim risks:` a strong win-rate result can still miss whether intermediate states are semantically preserved.

- **[TokAlign: Efficient Vocabulary Adaptation via Token Alignment](https://arxiv.org/abs/2506.03523)**. `Core idea:` learn a one-to-one mapping between source and target token IDs, then rearrange embeddings and continue fine-tuning under the new vocabulary. `LatentWire ablation:` before training the bridge, try explicit vocabulary remapping and compare against a token-overlap baseline and a frozen-vocab control. `Telemetry:` mapping purity, overlap rate, post-remap perplexity, restoration steps, transfer accuracy. `Claim risks:` successful remapping mainly shows surface mismatch can be reduced; it does not prove the bridge generalizes to low-overlap pairs.

- **[zip2zip: Inference-Time Adaptive Vocabularies for Language Models via Token Compression](https://arxiv.org/abs/2506.01084)**. `Core idea:` dynamically compress token streams into reusable hypertokens at inference time. `LatentWire ablation:` insert adaptive compression before the bridge and compare fixed-token input with on-the-fly compressed input at equal wall-clock and byte budgets. `Telemetry:` sequence-length reduction, hypertoken reuse, compression overhead, latency, downstream accuracy. `Claim risks:` speedups can vanish once compression and embedding-creation overhead are counted.

- **[Length-MAX Tokenizer for Language Models](https://arxiv.org/abs/2511.20849)**. `Core idea:` optimize the vocabulary for shorter sequences, not just frequency coverage, so tokenization itself lowers compute and KV-cache cost. `LatentWire ablation:` retokenize source and target with a length-optimal tokenizer, then test whether bridge quality changes after controlling for sequence length and model size. `Telemetry:` tokens/char, OOV rate, latency, throughput, KV-cache bytes, downstream accuracy. `Claim risks:` token-count reductions do not by themselves prove a better interface; they may only reflect shorter sequences.

- **[FOCUS: Effective Embedding Initialization for New Tokenizers](https://arxiv.org/abs/2305.14481)**. `Core idea:` initialize a new tokenizer by copying overlapping token embeddings and composing the rest from semantically similar shared tokens. `LatentWire ablation:` use FOCUS-style initialization as the tokenizer-adaptation baseline before bridge training, especially when vocabularies partially overlap. `Telemetry:` initialization perplexity, restoration steps, overlap coverage, downstream accuracy, convergence speed. `Claim risks:` easier embedding initialization can mask the harder problem of preserving higher-layer alignment.

## Highest-priority LatentWire ablations

1. **Byte-level bridge vs token-level bridge.** Replace token-ID supervision with byte/patch supervision and hold the byte budget fixed.
2. **Vocabulary remap before bridge training.** Run TokAlign-style remapping first, then measure whether the bridge still needs heavy alignment machinery.
3. **Time-warped span alignment vs fixed-position pairing.** Test whether Soft-DTW/aligned-span losses recover mismatched sequence lengths better than naive token pairing.
4. **Adaptive hypertoken compression before the bridge.** Compare fixed tokenization against zip2zip-style compression while matching wall-clock and bytes.
5. **Length-optimal retokenization control.** Retokenize both sides with a shorter-sequence vocabulary and check whether any bridge gain survives the control.
