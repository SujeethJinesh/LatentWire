# Multimodal / Diffusion / Latent Interface References for LatentWire

Web check: 2026-04-21. This memo is for interface design only. The goal is to borrow mechanisms that can plausibly improve cross-model reasoning/communication: projector alignment, connector bottlenecks, query-based compression, diffusion/flow refinement, latent tokenization, and expert fusion.

## Primary sources

- **[BLIP-2: Bootstrapping Language-Image Pre-training with Frozen Image Encoders and Large Language Models](https://arxiv.org/abs/2301.12597)**. `Core idea:` use a lightweight Q-Former to query frozen image features and bridge them into a frozen LLM with minimal trainable parameters. `LatentWire ablation:` compare the current bridge against a Q-Former-style query bottleneck with matched output token budget. `Telemetry to log:` query entropy, token budget, bridge accuracy, repair rate, latency. `Claim risk:` gains may come from parameter isolation and query compression rather than stronger cross-model semantics.

- **[Flamingo: a Visual Language Model for Few-Shot Learning](https://arxiv.org/abs/2204.14198)**. `Core idea:` use a Perceiver Resampler plus gated cross-attention to compress variable visual input into a fixed set of latent tokens. `LatentWire ablation:` replace the bridge with a resampler bottleneck and compare fixed-token, variable-token, and gate-free variants. `Telemetry to log:` latent count, gate open rate, attention concentration, bytes per example, downstream accuracy. `Claim risk:` fixed latent compression may help only because it standardizes sequence length, not because it improves communication fidelity.

- **[Visual Instruction Tuning / LLaVA](https://arxiv.org/abs/2304.08485)**. `Core idea:` align a frozen vision encoder to an LLM with a simple trainable projector, then instruction-tune on multimodal traces. `LatentWire ablation:` test a plain linear projector against deeper projector stacks and against the current bridge under the same supervision. `Telemetry to log:` projector rank, representation cosine, calibration loss, answer accuracy, data efficiency. `Claim risk:` a simple projector can look strong if the task mostly needs surface alignment rather than real interface reasoning.

- **[Libra: Building Decoupled Vision System on Large Language Models](https://arxiv.org/abs/2405.10140)**. `Core idea:` decouple inner-modal modeling from cross-modal interaction with a routed visual expert and explicit bridge module. `LatentWire ablation:` add routed experts on the bridge path and compare them with a single shared bridge at the same parameter budget. `Telemetry to log:` expert utilization, routing entropy, modality-specific error, transfer accuracy, recovery cost. `Claim risk:` routed gains may reflect specialization capacity rather than better cross-model communication.

- **[ILRR: Inference-Time Steering Method for Masked Diffusion Language Models](https://arxiv.org/abs/2601.21647)**. `Core idea:` guide denoising by dynamically aligning internal activations to a reference sequence throughout the iterative refinement process. `LatentWire ablation:` add reference-aligned steering to repair loops and compare against unguided and scalar-gated repair. `Telemetry to log:` trajectory distance to reference, stepwise help rate, false-skip rate, stability across steps. `Claim risk:` steering can improve outputs without improving the underlying bridge or latent protocol.

- **[Latent Refinement Decoding: Enhancing Diffusion-Based Language Models by Refining Belief States](https://arxiv.org/abs/2510.11052)**. `Core idea:` preserve distributional uncertainty during early denoising, then finalize confident tokens with feedback rather than hard commitment. `LatentWire ablation:` keep bridge states as soft mixtures longer before committing to discrete repair actions. `Telemetry to log:` belief entropy, commit step, repair saved, missed-help rate, token/byte cost. `Claim risk:` better uncertainty retention may just delay commitment; it may not solve representation mismatch.

- **[Progressive Refinement Regulation for Accelerating Diffusion Language Model Decoding](https://arxiv.org/abs/2603.04514)**. `Core idea:` regulate refinement with token-wise convergence signals and trajectory-grounded control rather than uniform step updates. `LatentWire ablation:` use step-specific stop/continue rules for repair or refinement rather than a single global threshold. `Telemetry to log:` convergence score, step count, early-stop rate, accuracy-vs-step curve, repair waste. `Claim risk:` if the controller is learned on the same task family, the result may be a control-policy win rather than an interface win.

- **[Flow Matching in Latent Space](https://arxiv.org/abs/2307.08698)**. `Core idea:` learn generative dynamics in a pretrained latent space for better compute efficiency and scalability. `LatentWire ablation:` treat the interface state as a latent flow and compare a direct one-shot bridge with iterative flow-style updates. `Telemetry to log:` update magnitude, step count, latent drift, reconstruction error, accuracy under compute budget. `Claim risk:` flow benefits may come from smoother latent dynamics, not from improved semantic transfer.

- **[Diffusion Transformers with Representation Autoencoders](https://arxiv.org/abs/2510.11690)**. `Core idea:` replace brittle low-dimensional VAEs with semantically rich representation encoders and a lightweight decoder head. `LatentWire ablation:` swap the current latent basis for a representation-autoencoder-style latent and test whether the bridge becomes easier to align. `Telemetry to log:` latent semantic linearity, reconstruction quality, bridge rank, accuracy, calibration sensitivity. `Claim risk:` the gain may be mostly from a stronger latent representation, not from the communication mechanism itself.

- **[FuseMoE: Mixture-of-Experts Transformers for Fleximodal Fusion](https://arxiv.org/abs/2402.03226)**. `Core idea:` route multimodal inputs through expert mixtures with learned gating so fusion can be sparse and modality-aware. `LatentWire ablation:` add sparse expert fusion to the bridge and compare it against a dense shared adapter at matched bytes. `Telemetry to log:` expert load balance, routing entropy, per-expert utility, bytes per route, accuracy. `Claim risk:` expert specialization can hide whether the bridge itself became better or merely more overparameterized.

- **[Sparse-Dense Mixture of Experts Adapter for Multi-Modal Tracking](https://arxiv.org/abs/2603.13719)**. `Core idea:` combine sparse and dense expert adapters so modality-specific and shared information are both modeled. `LatentWire ablation:` split the bridge into a sparse modality-specific path plus a dense shared path and compare it with single-path adapters. `Telemetry to log:` sparse/dense utilization, modality-specific error, transfer accuracy, dead-expert rate. `Claim risk:` the adapter split may help optimization without yielding interpretable cross-model communication.

## Highest-priority LatentWire ablations

1. **Q-Former / perceiver bottleneck.** Replace the current bridge with a fixed-query compressor and compare token budget, repair rate, and calibration.
2. **Simple projector vs routed expert bridge.** Test plain MLP projection against routed or expert-mixed connectors under the same byte budget.
3. **Soft belief-state refinement.** Keep bridge outputs probabilistic for longer, then commit with a local convergence rule.
4. **Trajectory-guided repair control.** Use per-step convergence signals to stop or continue repair instead of a global threshold.
5. **Latent-space flow bridge.** Compare one-shot transfer with iterative flow-style latent updates and log accuracy-vs-step curves.
