# Shared Feature Dictionary / Interface References for LatentWire

Web check: 2026-04-21. This memo prioritizes primary sources on sparse autoencoders, universal features across models, dictionary learning, transcoders / crosscoders, representation steering and interventions, and model merging symmetries. The common LatentWire question is whether we can communicate through a learned shared feature basis rather than through raw hidden states or text.

## Primary sources

- **[Towards Monosemanticity: Decomposing Language Models With Dictionary Learning](https://transformer-circuits.pub/2023/monosemantic-features/)**. `Math/interface idea:` learn an overcomplete sparse dictionary `x \approx W_d f(x)` so meaning is mediated by sparse features instead of neurons. `LatentWire ablation:` replace the current bridge with SAE latents as the shared interface, then compare raw residual transport, SAE-space transport, and SAE-space transport after feature pruning. `Telemetry:` feature sparsity, dead-feature rate, reconstruction loss, feature-logit attribution, monosemanticity proxy. `Claim risks:` a nicer dictionary is not proof of causal correctness; apparent interpretability can depend on SAE width, layer choice, and training recipe.

- **[Scaling Monosemanticity: Extracting Interpretable Features from Claude 3 Sonnet](https://transformer-circuits.pub/2024/scaling-monosemanticity/)**. `Math/interface idea:` decompose residual streams into feature reconstruction plus residual error, then steer by adding or removing individual feature directions. `LatentWire ablation:` use feature-level steering vectors as the bridge payload and compare them with raw hidden-state offsets and task-vector edits. `Telemetry:` feature-steering success rate, logit delta, residual norm, steering stability across prompts, side-effect rate on unrelated tasks. `Claim risks:` steering vectors can look reusable while actually being prompt-family specific, and the feature label may sit on a correlated direction rather than a clean concept.

- **[Quantifying Feature Space Universality Across Large Language Models via Sparse Autoencoders](https://arxiv.org/abs/2410.06981)**. `Math/interface idea:` paired SAE dictionaries from different models can be compared under rotation-invariant similarity, suggesting that feature spaces may align even when individual feature IDs do not. `LatentWire ablation:` train SAEs on two source/target models, align the feature dictionaries, and test whether the bridge can operate in a shared SAE basis rather than a model-specific basis. `Telemetry:` cross-model feature-correlation matrix, shared-space CKA/SVCCA, matched-feature cosine, transfer accuracy, basis-alignment residual. `Claim risks:` universality evidence can collapse if the models, layers, or domains are too similar; the result may describe a narrow family of checkpoints rather than a general shared interface.

- **[Transferring Linear Features Across Language Models With Model Stitching](https://arxiv.org/abs/2506.06609)**. `Math/interface idea:` fit an affine map between residual streams and reuse it to transfer SAE weights, probes, and steering vectors across model scales. `LatentWire ablation:` initialize a small bridge on a smaller donor model, stitch it into the larger target, and compare against training the bridge from scratch on the target. `Telemetry:` stitching residual, SAE transfer reconstruction loss, probe transfer accuracy, steering transfer fidelity, FLOPs to convergence. `Claim risks:` a good affine map may reflect shared low-level geometry rather than a genuinely model-agnostic interface, and gains may shrink when tokenizers or architectures diverge.

- **[Transcoders Beat Sparse Autoencoders for Interpretability](https://arxiv.org/abs/2501.18823)**. `Math/interface idea:` instead of reconstructing the input activation, a transcoder predicts the output of a component from its input, so the dictionary is tied to computation rather than just representation. `LatentWire ablation:` compare SAE-based bridges with transcoder-based bridges for the same model pair, especially when the goal is to preserve downstream behavior, not just encode activations. `Telemetry:` reconstruction loss, downstream task fidelity, feature-intervention effect size, feature density, interpretability ratings. `Claim risks:` transcoders can be more faithful to a local module while being less reusable as a general communication channel; better interpretability may come from output supervision rather than a better shared basis.

- **[Circuit Tracing: Revealing Computational Graphs in Language Models](https://transformer-circuits.pub/2025/attribution-graphs/methods.html)**. `Math/interface idea:` cross-layer transcoders create a local replacement model whose feature nodes can be traced through a graph of linear attributions, making multi-layer feature flow explicit. `LatentWire ablation:` use a cross-layer transcoder as the communication interface and compare a single-layer bridge, a multi-layer bridge, and a graph-traced bridge with constrained patching. `Telemetry:` edge attribution mass, graph sparsity, supernode size, intervention effect size, attention-frozen gap, faithfulness under patching. `Claim risks:` the graph is a local surrogate, so it may miss QK-mediated effects or second-order interactions that matter for end-to-end communication.

- **[Sparse Crosscoders for Cross-Layer Features and Model Diffing](https://transformer-circuits.pub/2024/crosscoders/)**. `Math/interface idea:` a crosscoder jointly learns a shared sparse dictionary over multiple layers or models, with decoder weights that can write to each target space. `LatentWire ablation:` train a shared crosscoder on source and target activations and compare it with separate per-model SAEs, especially when you want one interface that can hop across layers or across models. `Telemetry:` shared-feature fraction, per-model decoder norms, exclusive-feature density, cross-model reconstruction error, feature cosine alignment. `Claim risks:` shared-feature pressure can force an artificial consensus, and exclusive features may become denser simply because capacity is limited.

- **[ReFT: Representation Finetuning for Language Models](https://arxiv.org/abs/2404.03592)**. `Math/interface idea:` learn small interventions on hidden representations while freezing the base model, often as low-rank edits in a subspace of activations. `LatentWire ablation:` compare feature-space transport with learned hidden-state interventions at the same sites, then test whether the bridge needs a shared dictionary or just a good intervention subspace. `Telemetry:` intervention norm, rank used, layer-wise activation shift, parameter efficiency, accuracy vs edit magnitude. `Claim risks:` a strong ReFT result can be a task-specific intervention result rather than evidence of a reusable cross-model interface, and inference-time edits can hide communication cost.

- **[Git Re-Basin: Merging Models modulo Permutation Symmetries](https://arxiv.org/abs/2209.04836)**. `Math/interface idea:` hidden-unit permutation symmetry can be fixed by aligning units before weight-space merging, converting an apparently mismatched pair of models into a common coordinate system. `LatentWire ablation:` align feature dictionaries or bridge modules under permutation and orthogonal symmetries before merging or transferring them, then compare with naive averaging. `Telemetry:` assignment entropy, permutation stability, merge loss barrier, post-alignment cosine, functional delta after merge. `Claim risks:` permutation alignment fixes only one symmetry class; if the real mismatch is nonlinear or feature-level, a successful re-basing can still leave the bridge semantically wrong.

- **[TIES-Merging: Resolving Interference When Merging Models](https://arxiv.org/abs/2306.01708)**. `Math/interface idea:` resolve sign conflicts and drop low-change weights before merging, so the merged point keeps only aligned parameter updates. `LatentWire ablation:` test whether shared feature dictionaries need sign canonicalization or selective sparsification before being used as an interface across models. `Telemetry:` sign-conflict rate, trimmed-weight fraction, merged-task accuracy, conflict-resolution gain, per-layer interference score. `Claim risks:` sign alignment is a useful heuristic, but it may not generalize across architectures or capture more complex feature-space misalignments.

- **[Activation-Informed Merging of Large Language Models](https://arxiv.org/abs/2502.02421)**. `Math/interface idea:` use activation-space statistics to weight which parameters matter during merging, rather than treating all layers as equally mergeable. `LatentWire ablation:` add activation-informed weights to bridge or dictionary merging, and compare against uniform merge weights and simple task arithmetic. `Telemetry:` activation-guided merge weights, calibration-set loss, merged accuracy, robustness under distribution shift, layer importance variance. `Claim risks:` an activation-calibrated merge can overfit the calibration set or the donor pair, and may not indicate a stable shared feature geometry.

- **[Model soups: averaging weights of multiple fine-tuned models improves accuracy without increasing inference time](https://arxiv.org/abs/2203.05482)**. `Math/interface idea:` if fine-tuned checkpoints lie in one low-error basin, simple weight averaging can behave like a cheap interface over model variants. `LatentWire ablation:` use soup-style averaging as the dumb baseline for any bridge, adapter, or shared-dictionary training run before arguing for a more structured interface. `Telemetry:` soup accuracy, basin connectivity, loss barrier, calibration, OOD accuracy, compute-normalized gain. `Claim risks:` model soups are only informative when the checkpoints are already close in function and parameter space; outside that regime, averaging is not a meaningful communication primitive.

## Highest-priority shared-dictionary / interface ablations

1. **SAE basis vs raw residual bridge.** Train the bridge in SAE space and compare against raw hidden-state transport under matched bytes and FLOPs.
2. **Shared crosscoder vs per-model SAEs.** Force one dictionary to explain both models, then measure whether shared features stay sparse and causal.
3. **Transcoder bridge vs SAE bridge.** Decide whether the interface should reconstruct activations or predict module outputs, and score both on downstream fidelity.
4. **Feature steering vs learned intervention.** Compare explicit feature-direction edits with LoReFT-style representation interventions at the same layers.
5. **Symmetry-aware merge vs naive merge.** Canonicalize permutation, sign, and activation symmetries before merging bridge weights, then see whether the shared basis becomes more stable.
