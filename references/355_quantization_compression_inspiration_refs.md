# Quantization / Compression Inspiration for LatentWire

Web check: 2026-04-21. This memo is for bridge and latent-interface ablations, not general model compression. The goal is to borrow the *mechanisms* that matter for cross-model communication: saliency, outlier handling, bit allocation, rotations, codebooks, and KV-state compression.

## Primary sources

- **[AWQ: Activation-aware Weight Quantization for LLM Compression and Acceleration](https://arxiv.org/abs/2306.00978)**. `Core idea:` use activation statistics to protect salient weight channels and reduce quantization error with an equivalent scaling transform. `LatentWire ablation:` protect the highest-saliency bridge dimensions or atoms with higher precision, and compare against uniform bridge quantization. `Telemetry to log:` saliency histogram, protected-byte share, reconstruction error, task accuracy, calibration sensitivity. `Claim risk:` gains may come from preserving a tiny high-precision subset rather than from a better communication interface.

- **[GPTQ: Accurate Post-Training Quantization for Generative Pre-trained Transformers](https://arxiv.org/abs/2210.17323)**. `Core idea:` one-shot second-order quantization minimizes output error efficiently during post-training compression. `LatentWire ablation:` use Hessian-/curvature-aware scoring to choose which latent bridge coordinates or route atoms get high precision. `Telemetry to log:` curvature proxy, layerwise error, bit allocation, downstream accuracy, repair rate under compression. `Claim risk:` improvements may reflect better local error fitting, not better cross-model alignment.

- **[EXL2 quantization in ExLlamaV2](https://github.com/turboderp-org/exllamav2#exl2-quantization)**. `Core idea:` mixed-bit allocation per layer / per matrix using calibration error search to meet an average bitrate target. `LatentWire ablation:` mixed-bit latent bridge where each layer/head/atom gets an assigned bitwidth under a fixed average budget. `Telemetry to log:` bit histogram, average bitrate, max local error, calibration loss, task accuracy, latency. `Claim risk:` could mainly be a search-budget story; mixed-bit gains are not necessarily transferable to learned communication.

- **[SmoothQuant: Accurate and Efficient Post-Training Quantization for Large Language Models](https://arxiv.org/abs/2211.10438)**. `Core idea:` migrate activation outliers into weights via an equivalent transform so W8A8 becomes accurate and hardware-friendly. `LatentWire ablation:` smooth bridge activations before compression and compare to an unsmoothed control at matched bytes. `Telemetry to log:` activation outlier norms, smoothing scale, quantization error, accuracy, throughput / bytes. `Claim risk:` smoothing may just hide outliers rather than improve the bridge representation itself.

- **[QuaRot: Outlier-Free 4-Bit Inference in Rotated LLMs](https://arxiv.org/abs/2404.00456)**. `Core idea:` apply computationally invariant rotations that reduce outliers across hidden states, FFN activations, attention, and KV cache. `LatentWire ablation:` rotate latent bridge bases before compression, then test whether outlier-heavy coordinates become easier to transmit. `Telemetry to log:` outlier mass before/after rotation, reconstruction error, cross-model transfer, long-sequence stability. `Claim risk:` rotation benefits may depend on the full Transformer invariances and may not survive a smaller bridge-only setup.

- **[SpinQuant: LLM quantization with learned rotations](https://arxiv.org/abs/2405.16406)**. `Core idea:` learn rotation matrices that make quantization easier; rotation choice itself can change zero-shot quality materially. `LatentWire ablation:` learn a bridge rotation/alignment matrix before quantizing the interface, and compare it to random / Hadamard / identity rotations. `Telemetry to log:` rotation quality, outlier reduction, task accuracy, calibration loss, stability across seeds. `Claim risk:` a learned rotation can absorb most of the gain and obscure whether the communication protocol improved.

- **[KIVI: A Tuning-Free Asymmetric 2bit Quantization for KV Cache](https://arxiv.org/abs/2402.02750)**. `Core idea:` quantize KV cache asymmetrically with per-channel keys and per-token values to cut memory while keeping quality. `LatentWire ablation:` compress bridge-state history like a KV cache, then compare per-token vs per-channel vs asymmetric storage. `Telemetry to log:` cache bytes, retrieval misses, attention drift, long-context accuracy, decode throughput. `Claim risk:` memory wins may come from cache-specific asymmetry that does not improve latent transfer quality.

- **[KVQuant: Towards 10 Million Context Length LLM Inference with KV Cache Quantization](https://arxiv.org/abs/2401.18079)**. `Core idea:` combine per-channel key quantization, pre-RoPE quantization, non-uniform datatypes, and sparse outlier handling for long-context KV compression. `LatentWire ablation:` test bridge-state compression with pre-transform quantization, non-uniform bitwidths, and explicit outlier buckets. `Telemetry to log:` per-layer sensitivity, pre/post-transform error, cache bytes, long-range retention, accuracy decay. `Claim risk:` long-context gains may not translate to short-horizon cross-model communication.

- **[BAQ: Efficient Bit Allocation Quantization for Large Language Models](https://arxiv.org/abs/2506.05664)**. `Core idea:` solve a sensitivity-driven convex bit-allocation problem so precision varies across components instead of being uniform. `LatentWire ablation:` allocate bits across bridge layers, heads, or atoms using a sensitivity proxy instead of a flat budget. `Telemetry to log:` allocated bits per component, sensitivity proxy, total bytes, task accuracy, calibration loss. `Claim risk:` a good bit-allocation objective may improve compression without revealing a better latent interface.

- **[SpQR: A Sparse-Quantized Representation for Near-Lossless LLM Weight Compression](https://arxiv.org/abs/2306.03078)**. `Core idea:` isolate outlier weights into a sparse high-precision side matrix and quantize the remainder aggressively. `LatentWire ablation:` split bridge parameters into a sparse high-precision “exception” path plus a low-bit bulk path. `Telemetry to log:` outlier ratio, sparse side-matrix size, recovery accuracy, reconstruction error, runtime cost. `Claim risk:` sparse exceptions can dominate the result and make the bridge look better than it really is.

- **[Extreme Compression of Large Language Models via Additive Quantization (AQLM)](https://arxiv.org/abs/2401.06118)**. `Core idea:` compress weights with learned additive codebooks and joint block optimization at very low bitrates. `LatentWire ablation:` replace a linear bridge with a codebook / additive-residual bridge and compare against low-rank and token interfaces. `Telemetry to log:` codebook usage entropy, dead-code rate, bytes per latent, accuracy, calibration fit. `Claim risk:` codebooks can overfit calibration data and may be hard to interpret as a communicative protocol.

## Highest-priority LatentWire ablations

1. **Mixed-bit latent bridge.** Use EXL2- or BAQ-style bit allocation across bridge layers/heads/atoms under a fixed average budget.
2. **Outlier-protected bridge.** Split bridge parameters into a low-bit bulk path and a sparse high-precision exception path, AWQ/SpQR-style.
3. **Rotation-before-compression.** Compare identity, Hadamard, random orthogonal, and learned rotations before quantizing bridge states.
4. **Asymmetric KV-style bridge memory.** Compress bridge history with KIVI/KVQuant-style per-channel / per-token asymmetry and outlier buckets.
5. **Codebook bridge.** Replace some bridge projections with additive or grouped codebooks and log code usage, dead codes, and calibration overfit.
