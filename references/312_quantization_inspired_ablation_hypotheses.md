# Quantization-Inspired Ablation Hypotheses for LatentWire

Primary-source memo for compression math that can be transferred into LatentWire experiments on KV/latent transport, selective routing, protected channels, and residual refinement.

## Source-to-hypothesis table

|---|---|---|---|---|
| **AWQ** | [paper](https://arxiv.org/abs/2306.00978) | Preserve a tiny set of activation-salient channels while aggressively compressing the rest; the salient set is defined by activation statistics, not raw weight magnitude. | Add an activation-saliency selector for KV/latent transport: reserve a protected channel budget for the top `p%` saliency channels and compare against uniform bit allocation under matched byte budget. Metric: task accuracy vs transported bytes, plus protected-channel retention rate. | Guard against overfitting saliency to the calibration set; validate on held-out prompts and on both source and target models. |
| **GPTQ / OBQ** | [GPTQ paper](https://arxiv.org/abs/2210.17323), [OBQ / OBC paper](https://arxiv.org/abs/2208.11580) | Greedy second-order error compensation: after each quantization step, update remaining coordinates to cancel the local reconstruction error. | For latent transport, quantize route/value/residual blocks sequentially and apply local error compensation to the remaining blocks. Metric: reconstruction error after each stage, final answer accuracy, and sensitivity to quantization order. | Avoid claiming true second-order optimality unless the transport objective is explicit; keep the metric tied to downstream task loss or latent reconstruction loss. |
| **EXL2 / ExLlamaV2** | [repo](https://github.com/turboderp-org/exllamav2), [EXL2 docs in repo](https://github.com/turboderp-org/exllamav2#exl2-quantization) | Mixed-bit allocation per layer / per matrix, chosen by measuring reconstruction error under a target average bitrate. The important idea is not a single bitwidth, but adaptive rate allocation. | Sweep per-layer and per-route bit budgets for source KV and latent messages. Keep a global byte budget fixed, then compare uniform 4-bit, mixed-bit, and adaptive-bitrate schedules. Metric: accuracy per byte, layerwise error, and budget utilization. | Do not compare mixed-bit methods without a strict byte-matched baseline; the only meaningful result is accuracy at equal transport cost. |
| **KIVI** | [paper](https://arxiv.org/abs/2402.02750) | Asymmetric KV quantization: treat keys and values differently because their sensitivity profiles differ. | Split LatentWire communication into asymmetric key-path and value-path compression. Sweep `K_bits != V_bits`, and test whether routing entropy changes more on one side than the other. Metric: answer accuracy, retrieval accuracy, and K/V reconstruction asymmetry. | Guardrail: enforce the same total bytes across symmetric and asymmetric settings; otherwise the comparison is confounded. |
| **KVQuant** | [paper](https://arxiv.org/abs/2401.18079), [code](https://github.com/SqueezeAILab/KVQuant) | Large-context KV compression works best when the quantizer is aware of attention structure and head-specific saliency. | Add head-specific KV budgets for transported caches or latent buffers. Compare uniform head budgets against attention-aware head budgets. Metric: long-context retrieval accuracy, headwise entropy, and preserved top-k attention mass. | Validate on long-context tasks where cache quality matters; short prompts can hide failures in the transport layer. |
| **KV-Compress** | [paper](https://arxiv.org/abs/2410.00161) | Variable compression rates per attention head; some heads can be compressed far more aggressively than others. | Allocate transport budget per head / per route using a learned or heuristic saliency score. Compare fixed-rate compression to head-adaptive compression at the same average byte budget. Metric: per-head recall, answer accuracy, and budget variance across heads. | Guard against a method that wins only by spending more bytes on a few lucky heads. Report exact per-head budgets. |
| **LCQ** | [paper](https://arxiv.org/abs/2405.20973) | Low-rank codebooks beat rank-one codebooks when the compressed object has structure that a single centroid cannot represent. | Replace a single latent codebook with low-rank residual codebooks for transport payloads. Compare rank-1, low-rank, and additive residual codebooks. Metric: reconstruction error, collision rate, and downstream accuracy. | Ensure the gain is not just extra parameter count; normalize by codebook storage and runtime. |
| **AQLM / additive quantization** | [paper](https://arxiv.org/abs/2401.06118), [code](https://github.com/vahe1994/AQLM) | Sum multiple small codebooks instead of one large codebook, then optimize the residual each stage. | Use multi-stage residual message packing: one coarse codebook plus one or more residual codebooks for latent/KV transport. Metric: codebook utilization, residual norm decay, and accuracy at fixed bits. | Watch for codebook collapse where later stages do all the work; report stagewise contribution curves. |
| **OWQ** | [paper](https://arxiv.org/abs/2306.02272), [code](https://github.com/xvyaward/owq) | Outliers deserve separate treatment: protect the few channels that dominate quantization error. | Add a protected-channel transport path for outlier-heavy latent dimensions, with the rest compressed aggressively. Compare no-protection, hard-protection, and learned protection. Metric: outlier recovery rate and failure-case accuracy. | Guard against leakage of extra capacity through the protected path; keep the protected fraction fixed and small. |
| **TurboQuant** | [paper](https://arxiv.org/abs/2504.19874) | Geometric reparameterization can make quantization nearly distortion-optimal; the math is about better coordinates, not just lower precision. | Test a geometry-preserving latent transform before compression: rotation / polar / orthogonalization, then quantize in the transformed basis. Metric: reconstruction loss after transform, answer accuracy, and sensitivity to basis choice. | Do not overclaim that a coordinate transform is universally good; require ablation against identity and random orthogonal bases. |
| **TurboAngle** | [paper](https://arxiv.org/abs/2603.27467) | Angle quantization can preserve direction better than naive scalar quantization when the signal lives on a sphere-like manifold. | For latent transport, separate magnitude and direction, quantize them differently, and test whether directional preservation is the main determinant of success. Metric: cosine similarity, norm error, and downstream accuracy. | Guard against mixing coordinate-system gains with actual semantic transport; report both angle and norm metrics. |
| **InnerQ** | [paper](https://arxiv.org/abs/2602.23200) | Hardware-aware KV compression should optimize the quantization/dequantization path, not only the raw error metric. | Add an end-to-end cost model for transport: compression ratio, decode overhead, and total wall-clock, then compare methods at equal latency. Metric: accuracy per millisecond and bytes per token. | Do not let a purely theoretical compression gain hide a slow decode path; always report end-to-end latency. |

Metric: success rate under heterogeneous source/target pairs, compression ratio, and stability across model families. | Keep the comparison honest by fixing the same source-target pair and the same communication budget. |

## Ablation priorities that follow from these sources

1. **Asymmetric key/value budgets**: sweep `K_bits != V_bits` and log whether keys need more fidelity than values in cross-model transport.
2. **Protected-channel routing**: reserve a small outlier path for activation-salient dimensions and measure whether it carries most of the recoverable signal.
3. **Residual codebook transport**: compare single codebook vs multi-stage residual packing at equal bytes.
4. **Geometry-preserving transforms**: test identity, random orthogonal, learned orthogonal, and polar/angle-style bases before quantization.
5. **Head-adaptive compression**: allocate more bits to the attention heads that carry the highest preserved mass, then measure accuracy per byte and per millisecond.

## What to log in LatentWire

- Exact communication budget in bytes, bits, and milliseconds.
- Per-channel / per-head saliency, protected fraction, and protected-path utilization.
- Reconstruction loss after each compression stage.
- Cosine similarity, norm error, and top-k retention for geometric transforms.
- Downstream task accuracy under matched budget, not just raw compression error.
- Failure cases where a method wins on average but loses on long-context or outlier-heavy examples.

## Guardrails for interpretation

- Always match byte budget before comparing methods.
- Always compare against a no-transform / no-protection control.
- Always separate reconstruction metrics from task metrics.
- Always report whether gains come from geometry, outlier protection, or residual correction.
- Always include one heterogeneous source-target pair, one homogeneous pair, and one long-context pair.
