# Atlas-Alignment

- Title: `Atlas-Alignment: Making Interpretability Transferable Across Language Models`
- Date: 2025-10-31
- Link: https://arxiv.org/abs/2510.27413
- Why it matters here:
  - strongest recent reference for aligning models to a shared concept atlas rather than directly to one another
  - keeps the shared sparse / canonical interface lane alive even after dense transport and dense bridge variants plateau

Most transplantable mechanism:
- align model-specific activations to a shared semantic atlas so comparison happens in a canonical intermediate space instead of raw dense coordinates

Immediate use in our setting:
- use it as an anchor if the next method branch becomes a shared sparse dictionary or canonical concept-space bridge
