# Source-Private Evidence Packet Strict-Small Gate

Date: `2026-04-28`

## Cycle Start

1. Current ICLR readiness: not ready; this cycle clears a strict-small
   source-private benchmark contract, but the paper still needs a less
   protocol-shaped LLM/source-packet instantiation, seed repeats, and
   cross-family/model-pair confirmation.
2. Current paper story: source-private residual communication under decoder
   side information. The target has public prompt and candidate commitments;
   the source has private evidence; a rate-capped packet lets the target decode
   the right candidate.
3. Exact blocker to submission: show the same source-private gain when packets
   are produced/consumed by real model calls or a less synthetic task, not only
   by a deterministic syndrome decoder.
4. Current live branch: source-private evidence packet / candidate-syndrome
   decoder.
5. Highest-priority gate: strict-small budget sweep with target/no-source,
   matched-byte text, full-oracle, and source-destroying controls.
6. Scale-up rung: strict small gate.

## Subagent Guidance Integrated

- Reviewer: require target/no-source, target-wrapper, matched-byte text,
  answer-only, answer-masked, zero, shuffled, random, target-derived, exact-ID
  parity, byte accounting, and oracle rows. Synthetic benchmarks are defensible
  only if held-out IDs, public/private separation, and leakage controls are
  explicit.
- Planner: promote only if matched source beats target by at least `15` points
  and source-destroying controls stay flat; otherwise prune or repeat only if
  oracle headroom is large.
- Method scout: the first method should be the compact syndrome/evidence packet
  rather than a learned Q-gate. Learned Query-JEPA/Q-Former adapters remain
  second-stage once residual IDs are available.
- Harness audit: add exact-ID hashes, artifact hashes, explicit binary/text
  byte accounting, wrong-salt same-source control, and stronger tests.

## Implemented Gate

Script:

```bash
./venv_arm64/bin/python scripts/run_source_private_evidence_packet_strict_small.py \
  --examples 160 \
  --candidates 4 \
  --seed 28 \
  --budgets 2,4,8,16,32 \
  --output-dir results/source_private_evidence_packet_strict_small_20260428
```

Task:

- `160` frozen source-private QA examples.
- Target sees a natural-language public question and `4` candidate records with
  public cues and cryptographic commitments.
- Source sees private evidence containing the hidden witness key.
- Matched source sends a binary syndrome/digest prefix at `2`, `4`, `8`, `16`,
  or `32` bytes.
- Target decodes by matching the packet to candidate commitments.
- Public prior is intentionally nontrivial but weak: `40/160`, accuracy
  `0.250`.

Controls:

- target-only
- target-wrapper/no-source
- zero-source
- deterministic shuffled-source nonself
- random same-byte
- answer-only
- answer-masked
- target-derived sidecar
- wrong-salt same-source
- matched-byte structured text
- full structured text oracle
- full private evidence oracle

## Interpretation

This is a strict-small positive gate for the source-private candidate-syndrome
protocol. It proves the benchmark contract can produce a clean source-private
communication effect:

- the source has private evidence unavailable to the target
- the packet is rate-capped
- the target decodes with candidate side information
- same-byte random/shuffled/answer-only/answer-masked/target-derived/wrong-salt
  controls do not recover the gain
- matched-byte structured text does not explain the gain
- full text and full evidence oracles show headroom when the private key is
  transmitted without the strict byte cap

This is not yet an ICLR-ready paper claim. The benchmark is still
protocol-shaped and deterministic: the decoder knows the commitment protocol,
and packets are generated by code rather than an LLM/source agent. The result
promotes the candidate-syndrome branch, not the whole paper.

## Decision

Promote source-private evidence packets from smoke contract to live branch.
Do not yet promote Query-JEPA/Q-Former adapters. The next gate should replace
one deterministic part at a time:

1. keep the same frozen benchmark and have a source model produce packets under
   the same byte/token budgets, or
2. keep deterministic packets but move to a less cryptographic private-evidence
   task such as private tool/test-log handoff, then
3. run seed repeat and one model-pair/cross-family target.
